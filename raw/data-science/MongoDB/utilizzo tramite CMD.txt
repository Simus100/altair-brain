Possiamo usare il RB system tramite shell del sistema operativo. 

*I comandi del cmd verranno identificati con il simbolo % all'inizio.


PER CONNETTERCI:

% docker exec -it mongodb-container mongosh -u "root" -p "password"

(mongodb-container è il nome del container)


CASO D'USO:


SELEZIONE DB:

% use <nome database>  *selezione del DB

% show collections  *VIsualizza le collections del db


Poniamo il caso di avere un database chiamato "library" ocn due collections "books" e "users"

INSERIMENTO DA FILE JSON:

*i dati sono di esempio, sostituire le directory con quelle reali

1. copiare il file nela cartella di docker del DB:

% docker cp /user/mace/downloads/nomefile.json mongodb-container:/tmp/users.json

2. Importare il file nella colleciton:

% docker exec mongodb-container mongoimport --db library --collections users --file /tmp/users.json --jsonArray --username root --password password --authenticationDatabase admin


(possiamo usare "% mongoimport --help" per capire i comandi possibili).

% docker exec -it mongodb-container mongosh --username root --password password --authenticationDatabase admin

- possiamo anche aggiungere condizioni innestatE:

% db.users.find({grade_science: { $gt: 90 }})

* il precedente comando visualizza tutti gli oggetti della collection che hanno un valore all'attributo grade_science superiore a 90


% db.users.find({grade_science: { $lt: 90 }})

* il precedente comando visualizza tutti gli oggetti della collection che hanno un valore all'attributo grade_science inferiore a 90

*gt = greather then,  lt= less then, gte e lte (includono l'uguale a..)

- concatenazione di più condizioni


% db.users.find({grade_science: { $gte: 60, $lte: 90 }})

*il precedente comando visualizza i valori compresi tra 60 e 90



% db.users.find({
  grade_science: { $gte: 60, $lte: 90 },
  first_name: "Pippo"
})

*il precedente comando in visualizzazione più leggibile visualizza i valori compresi tra 60 e 90 che si chiamano Pippo


- inserimento di operatori

% db.users.find(
    {
       $or: [
          {favorite_color: "blue"},
          {favorite_color: "green"},
            ]
     }
)

* oltre a $or possiamo usare altri condizionali come $and



- ORDINAMENTO, LIMIT E SKIP


Prendiamo come esempio la query seguente e poi andiamo ad applicare dei filtri di ordinamento:


% db.users.find({
     $and: [
         { grade_science: {$gte: 60 } },
         {
       $or: [
          {favorite_color: "blue"},
          {favorite_color: "green"}
        ]
     }
  ]
},
  {first_name: 1, last_name: 1, favorite_color: 1, grade_science: 1}   
)

* questa riga:  {first_name: 1, last_name: 1, favorite_color: 1, grade_science: 1}   fa comparire nella query solo gli attributi selezionati.


quindi con il filtro:

$ db.users.find({
     $and: [
         { grade_science: {$gte: 60 } },
         {
       $or: [
          {favorite_color: "blue"},
          {favorite_color: "green"}
        ]
     }
  ]
},
  {first_name: 1, last_name: 1, favorite_color: 1, grade_science: 1}   
).sort({ last_name:1})


* con il ".sort({ last_name:1})" la query ordinerà glie elementi in ordine alfabetico di last_name.. possiamo anche indicare il parametro -1 per ordinare al contrario. Possiamo anche concatenare più ordinamenti: .sort({ last_name: 1, first_name: 1 })



In coda, dopo l'elemento sort possiamo anche aggiungere un limit per stabilire quanti elementi visualizzare:


%
    … {first_name: 1, last_name: 1, favorite_color: 1, grade_science: 1}   
).sort({ last_name:1}).limit(3)


Possiamo anche usare .skip() prima di .limit() per saltare degli elementi.

QUERY COMPLESSE: 

db.users.find({
 favorite_color: {$ne: "blue"}
})

*tutti gli utenti dove il colore è diverso da "blue"


db.users.find({
 favorite_color: {$in: ["blue", "yellow"]}
})

*utenti dove hanno le preferenze con dentro blue e yellow.


Esiste anche il comando $nin ovvero not in che esclude gli elementi che hanno quell'attributo.


db.users.uppdateOne ({user_id: 3}, {$set: {$mail: "abc@def.it}})

*Aggiunge il campo mail


db.users.find({user_id: 3}) 

*controlla che esita


db.users.find({
  email: {$exists: true}
})

*verifica che sia presente il campo selezionato


db.users.find({
 grade_marh: {$eq: null}
})

*cerca utenti con un valore null al parametro selezionato.

db.users.find({
 grade_marh: {$eq: null, $exists: False}

*cerca i valori null o l'assenza del campo

 first_name: {
  $regex: /^A/
 }
})

*Tutti gli utenti il cui nome inizia per A, per questa query vengono usate espressioni regolari, studiarle.
